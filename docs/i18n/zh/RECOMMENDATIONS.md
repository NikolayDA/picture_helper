## GitHub 开放议题 — 分诊状态（2026-07-12）

本次编辑前的实时状态：**#552**、**#549**、**#553** 与 **#550** 均已关闭。
剩余 **2** 个开放议题：**#245**（配额/计费阻塞）与 **#551**（Codex Security
Scan 策略决策）。

### 合理的组合

- **外部受阻：** #245 取决于 OpenAI 计费/配额——这是账户层面的操作，而非仓库
  PR。
- **安全扫描策略：** #551 在内容上与 #245 相关（同一个 Codex 扫描议题），但
  需要独立的策略决策（重新启用/停用/替换）。

评估：**相关性** = 对路线图/用户的重要性，**复杂度** = 预估实现工作量，
**模型/投入** = 建议由 Claude Code 实施时使用的 Claude 模型与推理投入等级。

| # | 标题 | 相关性 | 复杂度 | 模型/投入 | 建议的下一步 |
|---|------|--------|--------|-----------|--------------|
| [#551](https://github.com/NikolayDA/picture_helper/issues/551) | Codex Security Scan 策略决策（重新启用/停用/替换） | 🟡 中 | 🟡 中 | Sonnet 5 · 中 | **Needs refinement** – 需要在三个选项间做出明确取舍；建议：鉴于外部阻塞已持续数周且与 pip-audit/license/CI 存在冗余，选择方案 2（停用/禁用）。 |
| [#245](https://github.com/NikolayDA/picture_helper/issues/245) | Codex Security Scan「Quota exceeded」 | 🟡 中 | 🟢 低 | –（非代码任务） | **受阻（外部）** – 恢复 OpenAI 计费/配额是账户层面的操作，而非 PR。 |

### 接下来推荐（PR 顺序）

1. **#551** — 就扫描策略取得决策（与 #245 相关），随后相应调整 workflow。
2. **#245** — 保持外部受阻；仅在 OpenAI 配额恢复后手动验证。

*漂移提示：* 每次更新前重新核对实时开放议题数量，不要直接沿用旧值
（#542 → #549 出现过同样的偏差）。
