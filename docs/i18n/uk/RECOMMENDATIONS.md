## Поточний Стан (2026-06-04)

Активний список аналізу коду порожній. Ruff, mypy та локальний набір тестів
залишаються базовою перевіркою перед новими PR.

### Завершено З Останнього Review

- **N1/N2/N4/N5/N6/N7/N8** завершено: шляхи помилок, ліміт розміру,
  розширення, атомарне збереження, Qt-пакети CI, lazy import і docstring.
- **O2/O3/O4/O5/O6** реалізовано: Linux-пакети, release workflow, повна
  матриця, `ui_smoke` і платформні shortcuts.
- Знахідки **#163–#206** закрито в задокументованих PR і захищено
  регресійними тестами або перевірками CI.

### Ще Відкрито

- **O1 🟠 — Додаткові runtime-мови.** Німецьку й англійську можна перемикати
  в застосунку. Мови документації es/fr/uk/zh ще не є runtime locales;
  за потреби додати їх key-by-key у `bgremover.i18n` і покрити тестами.

### Рекомендований Порядок PR

1. **#247** — Висока: баг коректності/даних (застаріле crop-прямокутник дає прозорі пікселі); повністю окреслено, з регресійним тестом.
2. **#250** — Висока перед наступним release-тегом: примусити повний CI-gate через `needs`, звірити tag/версію, прибрати `|| true`.
3. **#251** — швидкий фікс пам'яті: порожня маска звільняє overlay-pixmap; точний патч в issue.
4. **#244** — прибирання мертвого коду (видалити `_zoom`, вирішити щодо `launch_worker`); малий низькоризиковий PR.
5. **#234** — не підвищувати версію за відсутності міграції та виправити тест, який зараз очікує протилежне.
6. **#248** — пріоритет Escape crop → ласо → зняти виділення; спільний контракт транзитного стану з #247, можна об'єднати.
7. **#232** — lazy exports PEP 562 з регресійним тестом імпорту.
8. **#249** — відкривати стартові шляхи й macOS-`QFileOpenEvent` через валідований шлях завантаження.
9. **#226** — документаційний fix у всіх шести мовах; чесно описати доступність releases.
10. **#245** — відновити білінг OpenAI на рівні акаунта; зробити scan-workflow стійким до вичерпання квоти і підняти `setup-node` до Node 24.
11. **#231** — спершу визначити модель скасування (підпроцес для назавжди заблокованих native-викликів), потім реалізувати.
12. **#235** — реалізувати спільний бюджет пам'яті undo/redo лише після визначення scope.
13. **#161/#203/#204** — закрити як `not planned`, якщо не буде надано конкретний шлях публікації або залежності.
