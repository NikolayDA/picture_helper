## Поточний стан (2026-07-06)

Активний список аналізу коду порожній. Ruff, mypy і локальний набір тестів
залишаються базою перед новими PR. Від знімка 2026-07-05 закрито
snapshot-fix Recommendations **#490**. Сьогоднішня перевірка епіків
редизайну (#413/#418/#424/#455/#463/#474/#483) виявила три нові, чітко
окреслені знахідки: **#499** (світла схема ще не 1:1 з прототипом), **#500**
(скрипт скриншотів зламано після редизайну, блокує #432) та **#501** (мертвий
до-редизайн-віджет `TopIconTab*`). GitHub наразі показує **14** відкритих
завдань.

### Нове з часу останнього огляду

- **#499 🟡 Баг/дизайн-система:** `theme.LIGHT` відхиляється від вбудованого
  CSS у `design/Prototyp A - Geführter Workflow.dc.html` за кількома токенами
  (`stepper`/`border`/`hairline`/`hover`/`card_border`/родина акценту) — той
  самий шаблон, що й вже завершене вирівнювання Dark Mode **#474–#480**, з
  тим самим тестовим каркасом (`tests/test_theme.py`), який уже є.
- **#500 🟠 Баг:** `scripts/generate_app_screenshots.py` шукає праву колонку
  через `findChild(QTabWidget)`, якої більше не існує з часів PR #412/#423
  (тепер це послідовність карток `Stepper`). Блокує **#432** (перестворити
  скриншоти) та будь-яку автоматизовану візуальну перевірку проти прототипу;
  досі без покриття тестами, чітко відтворюваний.
- **#501 🟢 Якість:** `TopIconTabBar`/`TopIconTabWidget` у `widgets.py` —
  мертві віджети з часів переходу на stepper (лишився лише lazy-export у
  `__init__.py` плюс згадка імпорту в `tests/test_package_imports.py`).
  Прибирання з низьким ризиком, без функціональних змін.

### Ще відкрито

- **O1 🟠 — Додаткові мови середовища виконання.** Німецька та англійська
  перемикаються; es/fr/uk/zh ще не є локалями виконання. Збігається із завданням
  редизайну **#430** — додати їх ключ-за-ключем у `bgremover.i18n` та покрити
  тестами.
- **O8 🟢 — Неточність прототипу: інструменти висоти лишаються заблокованими
  після генерації.** У `design/Prototyp A - Geführter Workflow.dc.html` дія
  «Створити карту висот із зображення» лише встановлює `heightGen`, не
  перемикаючи активний шар на роль `Höhe` — `heightDisabled` і далі залежить
  від попередньої ролі (знахідка рев'ю в PR #460). Стосується лише симуляції
  макета; реальний застосунок вже автоматично активує новий шар HEIGHT (#347).
