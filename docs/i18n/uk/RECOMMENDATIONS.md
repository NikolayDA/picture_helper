## Поточний стан (2026-07-03)

Активний список аналізу коду порожній. Ruff, mypy і локальний набір тестів
залишаються базою перед новими PR. Новина цього раунду: тріаж відкритих завдань
оновлено після follow-up щодо rail/zoom. До цього PR GitHub показує 14
відкритих завдань; після закриття **#461** лишається 13 завдань
roadmap/backlog.

### Виконано з часу останнього огляду

- **Стара база стабільна:** **N1/N2/N4/N5/N6/N7/N8** і **O2–O7** залишаються
  зробленими; епіки **#329/#344/#358/#384** (N9–N12) плюс виправлення експорту
  **#363** злиті, покриті тестами/CI та заархівовані.
- **Закрито з часу огляду 2026-06-25:** **#404**, **#406** та **#408**
  (PR #412) — знахідки щодо попереднього перегляду/мертвого коду/аудиту
  зроблено; `_derive_physical_size` більше не існує, а шлях рендеру
  деградує до COLOR за невідповідності розміру.
- **Ядро редизайну поставлено:** панель кроків/`stepper.py`, картковий інспектор,
  керована навігація, контекстні інструменти та токени дизайну
  (`ACCENT`/`CARD_STYLE`) прийшли через PR #412/#423 (рядки DE/EN,
  `tests/test_workflow.py`).
- **Хвилю rail/zoom завершено:** **#455/#456/#457/#458/#463/#464** прийшли через
  PR #466, а **#465** навмисно має статус `not_planned`. Цей follow-up PR
  закриває три пізні P2 з #466 (напрям масштабу, прив'язка до viewport,
  попередній перегляд height-dab) і оновлює **#461**.

### Ще відкрито

- **O1 🟠 — Додаткові мови середовища виконання.** Німецька та англійська
  перемикаються; es/fr/uk/zh ще не є локалями виконання. Збігається із завданням
  редизайну **#430** — додати їх ключ-за-ключем у `bgremover.i18n` та покрити
  тестами.
- **O8 🟢 — Неточність прототипу: інструменти висоти лишаються заблокованими
  після генерації.** У `design/Prototyp A - Geführter Workflow.dc.html` дія
  «Створити карту висот із зображення» лише встановлює `heightGen`, не
  перемикаючи активний шар на роль `Höhe` — `heightDisabled` і далі залежить
  від попередньої ролі (знахідка рев'ю в PR #460). Стосується лише симуляції
  макета; реальний застосунок вже автоматично активує новий шар HEIGHT (#347).

follow-up роботу й закрито в цьому PR. P2 з PR #460 лишається задокументованим
як mockup-only знахідка **O8**; реальний застосунок уже автоматично активує
шари HEIGHT.
