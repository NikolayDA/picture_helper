### Знахідки

- **N1 🟠 — Звільнення блокування чарівної палички на шляху помилки** (PR #142). Продовження «modest-shannon» B: під час зміни зображення `_load_image_async` скасовує flood fill, який тоді не емітує ні `finished`, ні `error`. Скидання блокування відбувалося лише на шляху успіху (`apply_loaded_image`): якщо завантаження зривалося, `_wand_busy` лишався активним, а паличка — заблокованою на старому зображенні. Нова тиха `reset_pending_wand()` просто поруч із `cancel_flood_fill()`.
- **N2 🟡 — Обмеження розміру при повороті** (PR #148). `rotate_image` (`image_ops.py`) повертає з `expand=True`; захист мегапікселів діяв лише під час завантаження (`Image.MAX_IMAGE_PIXELS`), а не на результаті — зображення трохи нижче ліміту могло роздутися до ~2× за ~45°. Тепер `rotated_size()` заздалегідь оцінює розширену bounding box; `apply_rotate` відхиляє результати понад ліміт зі статусним повідомленням.
- **N3 ➖ — Бюджет пам'яті історії** (уже покрито). `CanvasHistory` (`canvas_history.py`) давно забезпечує бюджет скасування через `_trim()`/`_UNDO_MEMORY_LIMIT`, а повтор обмежений `_REDO_MAX_ENTRIES`. Дій не потрібно.
- **N4 🟢 — Чесність розширення при збереженні** (PR #144). `save_image_file` мовчки писав байти PNG для невідомих розширень; тепер чітка відмова через `ValueError`, тоді як порожнє розширення лишається PNG за замовчуванням.
- **N5 🟡 — Атомарне збереження** (PR #144). Пряме записування в ціль руйнувало наявний файл у разі переривання. Тепер `mkstemp` → `os.replace` у цільовому каталозі (патерн `qt_plugins._copy_if_needed`), зі збереженням прав і прибиранням тимчасового файлу.
- **N6 🟡 — `libgl1` у повній матриці CI + тест дрейфу** (PR #143). Повна матриця не встановлювала `libgl1` (на відміну від інших джерел Qt-пакетів) → `import PyQt6` ризикував `libGL.so.1`. Додано; новий `test_ci_qt_packages.py` тримає всі чотири списки пакетів узгодженими.
- **N7 🟢 — Раннє імпортування** (PR #148). `workers.py` імпортував `rembg` (тягне onnxruntime) на рівні модуля; оскільки `main_window` завантажує `workers`, вартість імпорту сплачувалася вже на старті — навіть без використання ШІ. Тепер проба `find_spec` для `REMBG_AVAILABLE` плюс відкладений імпорт `rembg` лише у потоці воркера (warmup/перший клік ШІ).
- **N8 🟢 — Застарілий docstring `load_image`** (PR #143). Називав шлях drop синхронним викликачем, хоча перетягування давно працює асинхронно. Виправлено.

## Відкриті рекомендації

Покращення з другого аналізу, ще не реалізовані (продукт/процес):

- **O1 🟠 — Локалізація застосунку.** UI жорстко німецькою; немає i18n під час виконання (немає `QTranslator`/`tr()`), хоча документація існує п'ятьма мовами. Повідомлення статусу вже централізовані (`status_messages.py`). Поступово через Qt Linguist (`.ts`) або легку таблицю рядків за `QLocale`.
- **O2 🟡 — Linux-застосунок / пакування.** Немає бандла для Linux; запуск лише через `python -m bgremover` із venv. Інсталяційний пакет (AppImage/Flatpak/`.deb`) для **Raspberry Pi OS** і великих дистрибутивів (Debian/Ubuntu/Fedora) знижує поріг входу для не-розробників — аналогічно до macOS-бандла `.app`.
**✅ Виконано:** O4/O6 — перемикання інструментів однією клавішею (`W`/`B`/`E`/`L`) і платформні `Cmd`/`Ctrl`-підказки (PR #146, `test_tool_shortcuts.py`); O3 — повна матриця додатково щотижня через cron (PR #149); O5 — піднабір `ui_smoke` виконується в PR/Full CI, повна qtbot-сюїта лишається nightly (PR #149).

## План реалізації пакетами PR (з 2026-06-02)

- **PR 0 — Зміцнення коду (N2 + N7).** ✅ Виконано (PR #148). N2 — застосувати мегапіксельний запобіжник і до результату повороту (`rotated_size()` заздалегідь оцінює цільовий розмір, `apply_rotate` відхиляє результати понад ліміт зі статусним повідомленням); N7 — імпортувати `rembg` відкладено й перевіряти `REMBG_AVAILABLE` через `find_spec` (наявне оброблення збою warmup покриває дефектний бекенд).
- **PR 1 — Скорочення інструментів і підказки.** ✅ Виконано (PR #146). O4 + O6: перемикання однією клавішею (`W`/`B`/`E`/`L`), синхронізований checked-state toolbar, оновлені tooltips/README/посібник, регресійний тест wiring скорочень.
- **PR 2 — Раніше страхування CI.** ✅ Виконано (PR #149). O3 — повна матриця додатково щотижня (cron); O5 — піднабір `ui_smoke` у PR/Full CI, Nightly UI лишається повною сюїтою.
- **PR 3 — Основа i18n.** Підготувати O1: runtime-locale/fallback, поступово централізувати видимі рядки, німецьку лишити стабільним default.
- **PR 4 — Розгортання i18n.** Зробити O1 придатним: щонайменше англійська як runtime-мова, далі інші наявні мови документації, smoke checks для кожної locale.
- **PR 5 — Основа Linux-пакування.** Почати O2: вибрати цільовий артефакт (AppImage/`.deb`/Flatpak), desktop-файл/іконка/AppStream-метадані та Linux build smoke.
- **PR 6 — Розширення Linux-пакування.** Завершити O2: варіант для Raspberry Pi OS, опційний другий формат пакета та release workflow для Linux-артефактів.
