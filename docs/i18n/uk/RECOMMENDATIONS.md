Ruff, mypy та локальний набір тестів лишаються базою перед новими PR. Від минулого раунду повний цикл релізу v2.7.0 завершено:

1. **PR #670** (підняття версії `pyproject.toml`/`LICENSES.md`/`de.bgremover.app.metainfo.xml` до 2.7.0, перенесення CHANGELOG `[Unreleased]` → `[2.7.0]`, запис про іконку #667, узгодження RECOMMENDATIONS) злито (squash-коміт `6f103ed` на `main`).
2. Оскільки squash-злиття породило **новий** SHA коміту (відмінний від уже перевіреного `245f727`), повний gate кандидата **повторено** проти `6f103ed` — саме за правилом, задокументованим у `docs/history/RELEASE-2.6.0-candidate-gate.md` («примітка до #585»): повна CI-матриця (запуск [29989059554](https://github.com/NikolayDA/picture_helper/actions/runs/29989059554), зелений), збірка кандидата (запуск [29990198925](https://github.com/NikolayDA/picture_helper/actions/runs/29990198925), усі три платформи зелені, чисте сканування на секрети), апаратне приймання (запуск [29991314117](https://github.com/NikolayDA/picture_helper/actions/runs/29991314117): macOS arm64 ✅, Linux aarch64 ✅, x86_64 задокументовано призупинено, матриця приймання опублікована в #595).
3. Тег `v2.7.0` встановлено на `6f103ed` і відправлено (мав зробити сам власник репозиторію — git-проксі цієї сесії дозволяє пуш лише в призначену робочу гілку, не в теги чи `main`). Запуск workflow релізу [29998307692](https://github.com/NikolayDA/picture_helper/actions/runs/29998307692) зелений, job `Publish GitHub Release` успішний: **[v2.7.0](https://github.com/NikolayDA/picture_helper/releases/tag/v2.7.0)** опубліковано з усіма п'ятьма артефактами (AppImage + `.deb` для Linux x86_64/aarch64, `.dmg` для macOS arm64).

Крім того, від останнього знімку з'явилися дві нові автоматизовані задачі з аудиту:

- **#669** — справедливо вказує, що попередній live-стан цього документа був застарілим (усе ще перелічував #659/#660 як відкриті, бракувало #668). Виправлено цим оновленням.
- **#668** — `ANLEITUNG.md` досі посилається на осиротілий набір скриншотів від 2026-07-19 замість актуального від 2026-07-22 (супутня прогалина від #666); чиста гігієна репозиторію, без помилки змісту в самому посібнику.

Live-стан: **4** відкритих задачі (#669, #668, #656, #245) — усі чотири стосуються гігієни документації або суто зовнішні/операційні, жодного блокера коду.
