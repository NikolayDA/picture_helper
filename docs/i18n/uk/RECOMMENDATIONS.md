### Ще відкрито

- **O8 🟢 — Неточність прототипу:** інструменти висоти лишаються заблокованими в мокапі після генерації; стосується лише симуляції (#347).
- **N15 🟡 — Непротестоване з'єднання діалогу:** `MainWindow._open_ai_install_dialog` (`main_window.py:1566`) не має жодного окремого тесту, на відміну від структурно ідентичного сусіднього методу `_open_ai_model_dialog` (#597).
- **N16 🟡 — Непротестована не-RGBA-конвертація:** гілки не-RGBA у `pil_to_qpixmap`/`pil_to_numpy_readonly` (`image_utils.py:16`/`:43`) ніколи не викликаються з RGB-/палітровим/градаційним зображенням (#598).

### Доцільне групування

- **Release v2.6.0** (#580 → #583 → #584 → #585, строго послідовно): публікує вже готовий стан оновлень/керування ШІ з `main`; найвищий пріоритет через низький ризик і негайну користь для користувачів.
- **16-бітний конвеєр висоти** (#581 → #586 → #587 → {#588 ‖ #589} → #590): #586 (ADR) явно дозволено робити паралельно з релізом, але реалізація зі зміною схеми (#587+) починається лише після #585 (вимога scope-freeze із #580).
- **3D-попередній перегляд рельєфу** (#582 → #591 → #592 → #593 → #594 → #595): #591 додатково залежить від #586 (споживає той самий контракт HEIGHT) – тож #582 фактично йде за 16-бітним ланцюгом і є найбільшим блоком зусиль цього раунду.
- **#245/#551** лишаються пов'язаними (скан Codex: дія акаунту vs. принципове рішення).
- **#597/#598** — незалежні, повністю специфіковані прогалини покриття (ескіз тесту вже в задачі) – без ланцюга, без залежності від трьох епіків.

### Рекомендовано далі (порядок PR)

1. **#583** — спершу зайнятися scope-freeze v2.6.0: без відкритих залежностей, фіксує вже готову користувацьку цінність із низьким ризиком.
2. **#586** — почати 16-бітний ADR вже зараз, а не лише після #585: він блокує два цілі епіки (#581 напряму, #582 опосередковано через #591) і явно може йти паралельно з релізом.
3. **#551** — вирішити коротке, незалежне принципове питання; рекомендація й далі варіант 2 (демонтувати/вимкнути).
4. **#597 + #598** — найшвидший виграш у покритті цього раунду, обидва ескізи тестів уже в задачах; можна зробити одним спільним PR.
5. **#584/#585** та всі підзавдання 16-біт/3D послідовно йдуть за своїми залежностями – див. таблицю, додатковий тригер не потрібен.
