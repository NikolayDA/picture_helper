## Поточний стан (2026-07-22, після завершення приймання)

Ruff, mypy та локальний набір тестів лишаються базою перед новими PR. Новий dispatch `release-abnahme.yml`, запит на який був у минулому раунді, запущено (запуск [#4](https://github.com/NikolayDA/picture_helper/actions/runs/29908256619), коміт `9165c00`, після #657/#658) і він дав повністю зелену матрицю, окрім свідомо призупиненого рядка Linux x86_64. На цій основі всі чотири раніше заблоковані задачі були **окремо перевірені проти власних критеріїв приймання та закриті** — жодна задача не закрилася автоматично разом з іншою:

1. **#595** — усі пункти зі списку «досі відкрито» виконано; рядок x86_64 лишається «призупинено/декларовано відкритим», але за явним рішенням це не блокує закриття (за аналогією з власним критерієм приймання #639).
2. **#646** — п'ять із шести критеріїв уже виконано; знайдено реальний пробіл: `scripts/abnahme_vision_check.py`/`abnahme_aggregate.py` були повністю виключені з `make type`/`make check`, а строгий пробний запуск виявив справжню помилку `union-attr`. Виправлено та злито через PR #662 (коміт `f47445f`).
3. **#639** — після закриття #646 усі вісім дочірніх задач закрито; чекліст у тілі задачі узгоджено.
4. **#582** — усі п'ять дочірніх задач закрито; необхідне рішення щодо текстури як опційної цілі вже є в ADR, прогалину в README з аудиту від 2026-07-20 усунено, підтверджено зелений стан `make ui`.

Крім того, від минулого раунду з'явилися дві нові задачі з автоматизованих аудитів (**#659**, **#660**), поки без рішення.

Live-стан: **4** відкритих задачі.

### Результат перевірки

- **Стара база стабільна:** **N1/N2/N4/N5/N6/N7/N8**, **O1–O7** та все завершене від **2026-06-25** лишається зробленим.
- **#646/#639/#595/#582 тепер окремо перевірені й закриті** (без ефекту доміно автозакриття); єдиний реальний пробіл, знайдений по дорозі (строгість mypy для скриптів приймання, #646), виправлено та злито через PR #662.
- **#659/#660 нові й досі не вирішені:** #660 уже має готовий, але незлитий фікс на гілці `claude/festive-gates-4dkzds` (коміт `80b7aa0`); #659 — це чистий аналіз без змін коду, що пропонує два нові ідентифікатори знахідок (**N9**/**O8**), які досі очікують згоди власника.
- Крок, що лишився, тепер **не** стосується приймання — це відкриття/злиття PR для #660 та отримання рішення щодо знахідок, запропонованих у #659.
