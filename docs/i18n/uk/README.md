# BgRemover

Інструмент для редагування зображень на macOS для **видалення, заміни та редагування фону** — з автоматичним відокремленням об'єктів на основі ШІ, виділенням чарівною паличкою, пензлем/гумкою, обрізанням у різних форматах, обертанням, віддзеркаленням і заокругленням кутів.

## Функції

- **🤖 Видалення фону за допомогою ШІ** через [rembg](https://github.com/danielgatis/rembg) — один клік, і готово.
- **🪄 Чарівна паличка** — виділяє суміжні кольорові області за допомогою заливки (flood-fill) із повзунком допуску.
- **🖌 Пензель / гумка** — виділення можна домалювати або стерти вручну.
- **🎨 Замінити фон** — залити виділення будь-яким кольором або зробити його прозорим.
- **✂ Обрізання** із сіткою за правилом третин: коло, 1:1, 16:9, 4:3, 3:2, 2:1, 14:9, 9:16, 3:4.
- **⟲ Обертання** з кроком 90° або на довільний кут; **↔ Віддзеркалення** по горизонталі/вертикалі.
- **⬤ Заокруглення кутів** із регульованим радіусом.
- **↩ Історія** зі скасуванням і переходом до будь-якого попереднього кроку.
- **📥 Перетягування (Drag & Drop)** зображень безпосередньо у вікно.
- Збереження у форматах **PNG** (із прозорістю), **JPEG** (на білому фоні), **WebP** або **TIFF**.
- **⚙ Постійні налаштування** — стандартні каталоги та бажаний формат файлів зберігаються.

## Вимоги

- **macOS** (App-бандл, що додається, використовує специфічні для macOS інструменти, як-от `iconutil`)
- **Python 3.10 або новіший** (код використовує анотації типів PEP-604,
  як-от `QThread | None`, безпосередньо в сигнатурах — Python 3.9 не спрацює)
- Залежності (`PyQt6`, `Pillow`, `numpy`, опційно `rembg` для
  функції ШІ) встановлюються через `pyproject.toml`.

## Встановлення

**Рекомендовано (macOS): зібрати App-бандл.** Скрипт автоматично
створює ізольоване venv, встановлює всі залежності (включно з
`onnxruntime` для ШІ), коректно обробляє Apple Silicon і створює
самостійний `BgRemover.app`:

```bash
git clone https://github.com/NikolayDA/picture_helper.git
cd picture_helper
bash create_BgRemover_app.sh
```

Під час повідомлення про venv підтвердьте клавішею **Enter**. Після
цього запустіть `BgRemover.app` (у `~/Applications`) подвійним кліком —
функційно ідентичний до доданого **`BgRemover.command`**. Проєкт може
залишатися в `~/Documents` (застосунок збирається самостійно).

**Альтернативно напряму в терміналі** — на сучасному macOS у venv,
оскільки системний Python блокує `pip install` згідно з PEP 668:

```bash
python3 -m venv .venv && source .venv/bin/activate
python3 -m pip install -e ".[ai]"
python3 -m bgremover
```

`.[ai]` підтягує залежності ШІ (`rembg[cpu]` включно з `onnxruntime`);
без функції ШІ достатньо `python3 -m pip install -e .`.

**Linux:** App-бандла немає; застосунок працює через
прямий запуск з venv:

```bash
git clone https://github.com/NikolayDA/picture_helper.git
cd picture_helper
python3 -m venv .venv && source .venv/bin/activate
python3 -m pip install -e ".[ai]"
python3 -m bgremover
```

Попередньо потрібні деякі системні бібліотеки Qt — детальніше див.
**[INSTALL_LINUX.md](INSTALL_LINUX.md)**.

На **Raspberry Pi OS (Desktop)** це особливо просто — повністю без
venv/pip (PyQt6, Pillow, numpy як системні пакети через `apt`); див.
розділ про Raspberry Pi у **[INSTALL_LINUX.md](INSTALL_LINUX.md)**.

> (для тестування відкритих pull request) та **усуненням несправностей** —
> наведено в **[INSTALL_MAC.md](INSTALL_MAC.md)** (macOS) або
> **[INSTALL_LINUX.md](INSTALL_LINUX.md)** (Linux).

## Користування

1. **Відкрийте зображення** через `Файл → Відкрити` (⌘O) або перетягуванням у вікно.
2. **Зробіть виділення** чарівною паличкою, пензлем або гумкою (вкладка *🎯 Виділення*).
   - `Shift+клік` додає до виділення, `Ctrl+клік` віднімає.
3. **Відредагуйте фон** (вкладка *🖼 Фон*): зробіть прозорим або замініть колір — або одразу **ШІ** на панелі інструментів.
4. **Трансформуйте зображення** (вкладка *⟲ Транс.*): обертайте, віддзеркалюйте.
5. **Форма та обрізання** (вкладка *⬤ Форма*): заокругліть кути або обріжте за форматом — перемістіть/масштабуйте рамку, потім ✓ Застосувати.
6. **Збережіть** через `Файл → Зберегти` (⌘S) у форматі PNG, JPEG, WebP або TIFF.

### Налаштування

Через `Інструменти → Налаштування…` (⌘,) можна постійно зберегти три користувацькі налаштування:

| Налаштування | Опис |
|---|---|
| Стандартний каталог для відкриття | Початковий каталог діалогу відкриття; порожньо = останній використаний каталог |
| Стандартний каталог для експорту/збереження | Початковий каталог діалогу збереження; порожньо = останній використаний каталог |
| Бажаний формат файлів зображень | PNG, JPEG, WebP або TIFF — з'являється як перший варіант у діалозі збереження |

Налаштування постійно зберігаються через **QSettings** і автоматично відновлюються під час наступного запуску програми.

## Розробка та тести

```bash
git clone https://github.com/NikolayDA/picture_helper.git
cd picture_helper
pip install ".[test]"
make check
```

Набір тестів запускається headless (Qt-платформа `offscreen`) і перевіряє
операції із зображеннями, геометрію обрізання та логіку збереження. Для pull
request запускається легка GitHub PR CI (Ubuntu, Python 3.12, `make check`).
Повна матриця Linux/macOS з Python 3.10 та 3.12 запускається під час
публікації release або вручну. Повний процес тестування описано в
[TESTING.md](../../../TESTING.md).

Перевірка стилю коду та статична перевірка типів:

```bash
make lint
make type
```

## Архітектура (короткий огляд)

Починаючи з раунду 5, BgRemover — встановлюваний пакет (`bgremover/`,
запуск через `python -m bgremover` або console-script `bgremover`):

- **`ImageCanvas`** (QGraphicsView) зберігає стан зображення, маску виділення,
  стеки скасування/повторення та інструменти (чарівна паличка, пензель, ласо, обрізання).
- **`MainWindow`** будує панель інструментів, праву панель вкладок (чотири
  білдери `_build_tab_*`), меню і пов'язує все з полотном.
- **Worker'и** (`ImageLoadWorker`, `AIWorker`, `RembgWarmupWorker`) працюють у
  власних `QThread`'ах; `_launch_worker()` інкапсулює життєвий цикл потоку.
- Монотонний **лічильник версій** у полотні відкидає застарілі результати ШІ,
  якщо тим часом було завантажено інше зображення.
- Стек скасування обмежений не через `maxlen`, а через
  **ліміт пам'яті** (`_UNDO_MEMORY_LIMIT`); поточна
  сума байтів витісняє найстаріші записи.

## Відомі обмеження

- **Максимальний розмір зображення: 40 мегапікселів.** Більші зображення
  відхиляються з повідомленням у рядку стану. Виділення чарівною паличкою
  (flood-fill) працює синхронно в потоці UI; за цією межею навіть
  векторизований варіант помітно затримував би. Pillow додатково захищено
  від зображень типу «decompression bomb».
- **Збірка App-бандла** специфічна для macOS; під Linux/Windows
  застосунок працює через прямий запуск `python -m bgremover`.

## Файл журналу

Під час запуску програми створюється файл журналу `bgremover.log` у
специфічному для платформи каталозі даних застосунку
(macOS: `~/Library/Application Support/BgRemover/`,
Linux: `~/.local/share/BgRemover/`). Він містить трасування стека та
повідомлення про стан і є першим місцем для перевірки в разі проблем.
