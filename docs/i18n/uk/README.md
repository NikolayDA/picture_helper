# BgRemover

Інструмент для редагування зображень на macOS і Linux для **видалення, заміни та редагування фону** — з автоматичним відокремленням об'єктів на основі ШІ, виділенням чарівною паличкою, пензлем/гумкою, полігональним ласо, обрізанням у різних форматах, обертанням, віддзеркаленням і заокругленням кутів.

## Функції

- **🤖 Видалення фону за допомогою ШІ** через [rembg](https://github.com/danielgatis/rembg) — один клік, і готово.
- **🪄 Чарівна паличка** — виділяє суміжні кольорові області за допомогою заливки (flood-fill) із повзунком допуску.
- **🖌 Пензель / гумка** — виділення можна домалювати або стерти вручну.
- **➰ Полігональне ласо** — точно обмежує виділення за допомогою заданих кутових точок.
- **🎨 Замінити фон** — залити виділення будь-яким кольором або зробити його прозорим.
- **✂ Обрізання** із сіткою за правилом третин: коло, 1:1, 16:9, 4:3, 3:2, 2:1, 14:9, 9:16, 3:4.
- **⟲ Обертання** з кроком 90° або на довільний кут; **↔ Віддзеркалення** по горизонталі/вертикалі.
- **⬤ Заокруглення кутів** із регульованим радіусом.
- **↩ Історія** зі скасуванням і переходом до будь-якого попереднього кроку.
- **📥 Перетягування (Drag & Drop)** зображень безпосередньо у вікно.
- Збереження у форматах **PNG** (із прозорістю), **JPEG** (на білому фоні), **WebP** або **TIFF**.
- **⚙ Постійні налаштування** — стандартні каталоги та бажаний формат файлів зберігаються; у налаштуваннях можна знайти файл журналу та відкрити його каталог.

## Вимоги

- **macOS** або **Linux із графічним середовищем** (опційний App-бандл
  використовує специфічні для macOS інструменти, як-от `iconutil`)
- **Python 3.10 або новіший** (код використовує анотації типів PEP-604,
  як-от `QThread | None`, безпосередньо в сигнатурах — Python 3.9 не спрацює)
- Залежності (`PyQt6`, `Pillow`, `numpy`, опційно `rembg` для
  функції ШІ) встановлюються через `pyproject.toml`.

Для відтворюваного snapshot ШІ/app рекомендовано **Python 3.11 або
новіший**: деякі актуальні транзитивні залежності ШІ більше недоступні
під Python 3.10. Базовий застосунок без ШІ і надалі підтримує Python 3.10.

## Встановлення

**Рекомендовано (macOS): зібрати App-бандл.** Скрипт автоматично
створює ізольоване app-venv, намагається встановити залежності ШІ,
включно з `onnxruntime`, коректно обробляє Apple Silicon і створює
launcher `BgRemover.app`:

```bash
git clone https://github.com/NikolayDA/picture_helper.git
cd picture_helper
bash create_BgRemover_app.sh
```

Якщо app-venv створюється заново, підтвердьте повідомлення клавішею
**Enter**. Після цього запустіть `BgRemover.app` (у `~/Applications`)
подвійним кліком — функційно ідентичний до доданого
**`BgRemover.command`**. Launcher використовує окремо встановлене venv у
`~/Library/Application Support/BgRemover/venv`, тому проєкт може
залишатися в `~/Documents`. Водночас app і app-venv належать одне до
одного: файл `.app` не є портативним сам по собі. Якщо встановлення
залежностей ШІ не вдається, скрипт збирає придатний до використання
застосунок без ШІ.

Після оновлення або перемикання гілки знову виконайте
`bash create_BgRemover_app.sh`. Скрипт не-editable перевстановить поточний
checkout поверх наявного app-venv і перебудує launcher.

**Альтернативно напряму в терміналі** — на сучасному macOS у venv,
оскільки системний Python блокує `pip install` згідно з PEP 668:

```bash
python3 -m venv .venv && source .venv/bin/activate
python3 -m pip install --upgrade "pip>=26.1.2"
python3 -m pip install -c requirements/constraints.txt -e ".[ai]"
python3 -m bgremover
```

`.[ai]` підтягує залежності ШІ (`rembg[cpu]` включно з `onnxruntime`);
без функції ШІ достатньо `python3 -m pip install -c requirements/constraints.txt -e .`.

**Linux:** Для кінцевих користувачів рекомендовані release-артефакти:
портативний **AppImage** та встановлюваний **`.deb`** (обидва для x86_64 і
aarch64/Raspberry Pi OS). Деталі встановлення див. у
**[INSTALL_LINUX.md](INSTALL_LINUX.md)**, а деталі build/пакування — у
**[packaging/linux/README.md](../../../packaging/linux/README.md)**. Такі артефакти
доступні починаючи з **v2.3.0** — доти користуйтеся способом через venv нижче.

Прямий запуск із venv залишається найкращим шляхом для розробки,
тестування гілок і локальних змін:

```bash
git clone https://github.com/NikolayDA/picture_helper.git
cd picture_helper
python3 -m venv .venv && source .venv/bin/activate
python3 -m pip install --upgrade "pip>=26.1.2"
python3 -m pip install -c requirements/constraints.txt -e ".[ai]"
python3 -m bgremover
```

Перед запуском із venv потрібні деякі системні бібліотеки Qt — див.
**[INSTALL_LINUX.md](INSTALL_LINUX.md)**. На **Raspberry Pi OS (Desktop)**
також можна обійтися без venv/pip (PyQt6, Pillow, numpy як системні пакети
через `apt`); див. також **[INSTALL_LINUX.md](INSTALL_LINUX.md)**.

## Користування

1. **Відкрийте зображення** через `Файл → Відкрити` (⌘O) або перетягуванням у вікно.
2. **Зробіть виділення** чарівною паличкою, пензлем, гумкою або полігональним ласо (вкладка *🎯 Виділення*).
   - `Shift+клік` додає до виділення; `⌘+клік` (macOS) або `Ctrl+клік` (Linux) віднімає.
   - Перемикайте інструменти з клавіатури: `W` паличка, `B` пензель, `E` гумка, `L` ласо.
3. **Відредагуйте фон** (вкладка *🖼 Фон*): зробіть прозорим або замініть колір — або одразу **ШІ** на панелі інструментів.
4. **Трансформуйте зображення** (вкладка *⟲ Транс.*): обертайте, віддзеркалюйте.
5. **Форма та обрізання** (вкладка *⬤ Форма*): заокругліть кути або обріжте за форматом — перемістіть/масштабуйте рамку, потім ✓ Застосувати.
6. **Збережіть** через `Файл → Зберегти` (⌘S) у форматі PNG, JPEG, WebP або TIFF.

### Налаштування

Через `Інструменти → Налаштування…` (⌘,) можна керувати такими налаштуваннями:

| Налаштування | Опис |
|---|---|
| Стандартний каталог для відкриття | Початковий каталог діалогу відкриття; порожньо = останній використаний каталог |
| Стандартний каталог для експорту/збереження | Початковий каталог діалогу збереження; порожньо = останній використаний каталог |
| Бажаний формат файлів зображень | PNG, JPEG, WebP або TIFF — з'являється як перший варіант у діалозі збереження |
| Мова | Німецька або англійська; зміна набуває чинності після перезапуску |
| Файл журналу | Показує шлях до файлу журналу; кнопка «Відкрити каталог» відкриває каталог у файловому менеджері |

Директорії, бажаний формат і мова зберігаються через **QSettings** та
автоматично відновлюються під час наступного запуску програми.

## Розробка та тести

```bash
git clone https://github.com/NikolayDA/picture_helper.git
cd picture_helper
python3 -m venv .venv
source .venv/bin/activate
make pr-check
```

Набір тестів запускається headless (Qt-платформа `offscreen`) і перевіряє
операції із зображеннями, геометрію обрізання та логіку збереження. Для pull
request запускається легка GitHub PR CI (Ubuntu, Python 3.12, `make pr-check`).
Повна матриця Linux/macOS з Python 3.10, 3.11, 3.12 та 3.13 запускається як
release-gate: під час push тега версії робочий процес release викликає її
перед публікацією; крім того — щотижня (у неділю) та вручну. Усі
локальні/CI test-встановлення
використовують `requirements/constraints.txt`; за потреби шлях можна
перевизначити через `make PIP_CONSTRAINT=/шлях/до/файлу pr-check`.
Повний процес тестування описано в [TESTING.md](../../../TESTING.md).

Перевірка стилю коду та статична перевірка типів:

```bash
make lint
make type
```

## Архітектура (короткий огляд)

BgRemover — встановлюваний пакет (`bgremover/`, запуск через
`python -m bgremover` або console-script `bgremover`):

- **`ImageCanvas`** (QGraphicsView) зберігає стан зображення, маску виділення,
  стеки скасування/повторення та інструменти (чарівна паличка, пензель, ласо, обрізання).
- **`MainWindow`** будує панель інструментів, status/crop bar і пов'язує
  полотно, меню, праву панель і workers.
- **`right_panel`** будує чотири праві вкладки для вибору, фону,
  обертання/дзеркалення та форми/обрізання з набору callbacks.
- **`menu_actions`** будує рядок меню, actions і shortcuts; `MainWindow`
  лише передає callbacks.
- **`RecentFiles`** інкапсулює персистентність, дедуплікацію та адаптер
  меню «Відкрити нещодавні», тож `MainWindow` лише делегує шлях завантаження.
- **Worker'и** (`ImageLoadWorker`, `AIWorker`, `RembgWarmupWorker`,
  `FloodFillWorker`) працюють у
  власних `QThread`'ах; `WorkerController` інкапсулює старт, сильні
  посилання на worker-и, `deleteLater` і shutdown.
- Монотонний **лічильник версій** у полотні відкидає застарілі результати
  ШІ та flood-fill, якщо тим часом було завантажено інше зображення або
  змінено стан зображення.
- Стек скасування обмежений не через `maxlen`, а через
  **ліміт пам'яті** (`_UNDO_MEMORY_LIMIT`); поточна
  сума байтів витісняє найстаріші записи.

## Відомі обмеження

- **Максимальний розмір зображення: 40 мегапікселів.** Більші зображення
  відхиляються з повідомленням у рядку стану, щоб обмежити використання
  пам'яті та час обробки. Виділення чарівною паличкою (flood-fill)
  обчислюється асинхронно у власному `QThread`, тож інтерфейс залишається
  чутливим під час обчислення. Pillow додатково захищено від зображень
  типу «decompression bomb».
- **Збірка App-бандла** специфічна для macOS; під Linux застосунок працює
  через прямий запуск `python -m bgremover`. Windows наразі не входить до
  офіційно протестованої матриці.

## Файл журналу

Внутрішній logger застосунку використовує файл `bgremover.log` у каталозі
даних, визначеному Qt. Точний шлях залежить від платформи та конфігурації
Qt; з поточною конфігурацією macOS це
`~/Library/Application Support/BgRemover/BgRemover/bgremover.log`, а під
Linux файл міститься нижче `~/.local/share/`. Він містить повідомлення
виконання та трасування стека зареєстрованих помилок і створюється під час
першого запису в журнал.

Launcher macOS App-бандла додатково записує стартову діагностику в
`~/Library/Application Support/BgRemover/bgremover.log`.

Точний внутрішній шлях показано в `Інструменти → Налаштування… → Файл
журналу`; кнопка «Відкрити каталог» відкриває каталог безпосередньо у
файловому менеджері.
