# BgRemover – Посібник користувача

Цей посібник крок за кроком описує, як користуватися програмою
**BgRemover** — від першого відкриття зображення до збереження готового
результату. Він розрахований на користувачів без попереднього досвіду
редагування зображень.

> Примітки щодо **встановлення** навмисно не включено сюди — вони в
> [INSTALL_MAC.md](INSTALL_MAC.md) (macOS) та
> [INSTALL_LINUX.md](INSTALL_LINUX.md) (Linux). Цей посібник передбачає,
> що програму вже можна запустити.

## 1. Що вміє BgRemover?

BgRemover — інструмент редагування зображень для **видалення, заміни та
редагування фонів** — з додатковими функціями для простої оптимізації
зображень, шарів/проєктів і підготовки ресурсів для UV-друку. **Керований
6-кроковий робочий процес** (Відкрити → Вилучити фон → Корекція → Форма й
розміри → Рельєф і шари → Експорт) веде вас крізь редагування. Основні
функції:

- **Видалення фону за допомогою ШІ** – автоматичне видалення фону одним
  кліком.
- **Ручне виділення** за допомогою чарівної палички, пензля, гумки та
  полігонального ласо.
- **Замінити фон** – зробити виділення прозорим або заповнити будь-яким
  кольором.
- **Трансформація** – обертання (кроками по 90° або вільним кутом) і
  дзеркалення.
- **Форма та обрізання** – заокруглення кутів, обрізання до кола або
  фіксованого співвідношення сторін.
- **Оптимізація зображення** – налаштування яскравості, контрасту й
  насиченості, а також згладжування альфа-краю (feather).
- **Розмір і фізичні розміри** – зміна розміру в пікселях або задання розміру
  друку через міліметри й DPI (з підказкою про площу друку).
- **Шари та проєкти** – керування кількома шарами (колір/висота/глянець/
  загальний) і збереження та відкриття всього як проєкту `.bgrproj`.
- **Карти висот** – створення карти висот із зображення, її редагування за
  повзунками або безпосередньо пензлем і оптимізація.
- **2D-перегляд** – перевірка кольору, рельєфу, висоти та глянцю на екрані.
- **Експорт для EufyMake Studio** – створення ресурсів імпорту для UV-друку.
- **Історія** зі скасуванням/повторенням і переходом до будь-якого
  попереднього кроку редагування.
- **Збереження** у форматах PNG, JPEG, WebP або TIFF.

## 2. Вікно програми — огляд

![BgRemover – головне вікно після запуску](../../../app_screenshots/bgremover_complete_20260711_094027/01_main_empty.png)

*Головне вікно одразу після запуску: рядок меню вгорі, панель інструментів
ліворуч, полотно з шаховим візерунком прозорості по центру, крокова панель
над полотном, картковий інспектор праворуч (тут крок 1 «Відкрити») та рядок
стану внизу.*

Вікно поділено на п'ять зон:

```
┌─────────────────────────────────────────────────────────────┐
│ Рядок меню                                                    │
├──────────┬───────────────────────────────┬──────────────────┤
│          │      Крокова панель (6 кроків)                    │
│ Панель ──┼───────────────────────────────┼──────────────────┤
│  інстр.  │                               │  Картковий       │
│ (ліворуч)│           Полотно             │  інспектор       │
│          │      (зображення + виділення) │  (праворуч)      │
│          │                               │                  │
├──────────┴───────────────────────────────┴──────────────────┤
│ Рядок стану (підказки та повідомлення)                        │
└──────────────────────────────────────────────────────────────┘
```

| Зона | Призначення |
|---|---|
| **Рядок меню** (вгорі) | Файл, Проєкт, Редагування, Вигляд, Інструменти |
| **Крокова панель** (над полотном) | Шість кроків: Відкрити, Вилучити фон, Корекція, Форма й розміри, Рельєф і шари, Експорт |
| **Панель інструментів** (ліворуч) | Переміщення/масштаб, контекстні інструменти виділення/висоти, скасувати/повторити/тема |
| **Полотно** (по центру) | Показує зображення та поточне виділення; зумова пігулка внизу праворуч показує та регулює масштаб |
| **Картковий інспектор** (праворуч) | Заголовок із назвою/описом кроку, картки активного кроку, підвал із «Назад»/«Далі» |
| **Рядок стану** (внизу) | Підказки та повідомлення програми |

### Меню «Редагування», «Вигляд», «Проєкт» та «Інструменти»

Багато дій також доступні через рядок меню:

- **Редагування** – скасувати/повторити, поворот (90° ліворуч/праворуч/
  180°), горизонтальне/вертикальне дзеркалення, *Змінити розмір…*, а також
  зняти/інвертувати виділення та *Відновити оригінал*. Зручно, коли ви
  віддаєте перевагу меню перед панеллю інструментів або картковим
  інспектором.
- **Вигляд** – *Вписати у вікно* (⌘0), *Історія* (відкриває ту саму
  історію змін, що й раніше кнопка на панелі інструментів), підменю *Режим
  перегляду* (див. [розділ 12](#12-крок-6--експорт)) та *Світла тема* для
  перемикання кольорової схеми.
- **Проєкт** – *Новий проєкт*, *Відкрити проєкт…*, *Зберегти проєкт* /
  *…як…* (`.bgrproj`) та *Експортувати ресурси для EufyMake Studio…* (див.
  [розділ 11](#11-крок-5--рельєф-і-шари) і
  [розділ 12](#12-крок-6--експорт)).
- **Інструменти** – *Налаштування…* (див. [розділ 13](#13-налаштування)),
  *Перевірити оновлення…*, *Керувати моделлю ШІ…* та *Встановити
  видалення фону за допомогою ШІ…* (див.
  [розділ 7](#7-крок-2--вилучити-фон) і
  [розділ 18](#18-усунення-неполадок-і-файл-журналу)).

![Меню «Редагування»](../../../app_screenshots/bgremover_complete_20260711_094027/23_menu_edit.png)

*Меню «Редагування» об'єднує скасувати/повторити, поворот, дзеркалення та
дії з виділенням.*

### Крокова панель

Над полотном **крокова панель** веде через шість станцій: **Відкрити →
Вилучити фон → Корекція → Форма й розміри → Рельєф і шари → Експорт**. Клік
на вже досягнутому або розблокованому кроці одразу переходить до нього; без
завантаженого зображення кроки 2–6 залишаються заблокованими (вільний лише
крок 1). Виконані кроки позначені галочкою, активний крок виділено. Внизу
карткового інспектора кнопки **«← Назад»** та **«Далі: …»** ведуть крізь
процес; на останньому кроці кнопка натомість запускає
**«Експортувати ✓»** (збереження).

### Масштабування та вигляд

- **Масштабування:** **колесом миші** над полотном збільшуйте чи
  зменшуйте вигляд, або скористайтеся плаваючою **зумовою пігулкою** внизу
  праворуч на полотні (**−** / значення у відсотках / **+** / замок для
  фіксації масштабу).
- **Переміщення:** якщо зображення більше за вікно, перетягуйте його
  інструментом **Переміщення/Масштаб** (перетягування лівою кнопкою) або
  смугами прокручування.
- **Вписати:** `Вигляд → Вписати у вікно` (⌘0) знову вписує зображення
  повністю у вікно. Під час завантаження зображення це відбувається
  автоматично.

## 3. Швидкий старт за 5 кроків

Видаліть фон менш ніж за хвилину:

1. **Відкрити зображення** – на кроці *Відкрити* перетягніть зображення в
   поле для перетягування, скористайтеся `Файл → Відкрити` (⌘O / Ctrl+O)
   або перетягніть його безпосередньо на полотно.
2. **Запустити ШІ** – на кроці *Вилучити фон* угорі натисніть **«Вилучити
   фон (ШІ)»**. Фон видаляється автоматично.
3. **Доопрацювання (необов'язково)** – **гумкою** видаліть залишки
   виділення або **пензлем** додайте потрібне.
4. **Перевірка** – за потреби натисніть **Скасувати** (⌘Z), щоб відступити
   на крок назад.
5. **Зберегти** – на кроці *Експорт* виберіть формат **PNG** (зберігає
   прозорість) і натисніть **Зберегти**, або скористайтеся `Файл →
   Зберегти` (⌘S).

![Результат видалення фону за допомогою ШІ](../../../app_screenshots/bgremover_complete_20260711_094027/55_function_ai_result.png)

*Після одного кліку на «Вилучити фон (ШІ)» фон вирізається автоматично –
рядок стану повідомляє «Вилучення фону ШІ завершено», а вільні ділянки
показує шаховий візерунок.*

Наступні розділи пояснюють кожен крок докладно.

## 4. Крок 1 – Відкрити зображення

Є кілька способів завантажити зображення:

- **Поле для перетягування (крок 1):** перетягніть зображення з менеджера
  файлів безпосередньо на пунктирне поле в картковому інспекторі або
  клацніть по ньому, щоб відкрити діалог вибору файлу.
- **Меню:** `Файл → Відкрити…` (⌘O / Ctrl+O).
- **Перетягування на полотно:** перетягніть файл зображення безпосередньо
  на полотно. Якщо перетягнути кілька файлів, завантажується лише перше
  зображення.
- **Нещодавно відкриті:** `Файл → Нещодавно відкриті`, а також картка
  «Нещодавно відкриті» на кроці *Відкрити* (до трьох записів із мініатюрою)
  показують останні використані елементи. Це як зображення, так і
  **проєкти** `.bgrproj` (див. [розділ 11](#11-крок-5--рельєф-і-шари)); після
  кліку програма визначає тип і відкриває його відповідно.
- **Запуск зі шляхом до зображення:** якщо програму запущено зі шляхом до
  зображення — через **командний рядок** (`bgremover image.png`) або
  **Linux-ярлик робочого столу** (асоціація файлів) — вона завантажує це
  зображення безпосередньо під час запуску.
- **Відкриття через macOS Finder:** на macOS зображення також можна
  передати BgRemover **подвійним кліком**, через «Відкрити за допомогою…»
  або **асоціацію файлів** у Finder.

Усі ці шляхи використовують той самий **перевірений асинхронний шлях
завантаження**: застосовуються однакові перевірки формату й розміру, а
великі зображення завантажуються у фоновому режимі — рядок стану показує
прогрес. Після завантаження крокова панель автоматично розблоковує
наступний крок.

![Меню «Файл»](../../../app_screenshots/bgremover_complete_20260711_094027/20_menu_file.png)

*Меню «Файл» містить Відкрити (⌘O), «Нещодавно відкриті», Зберегти (⌘S) та
Зберегти як… (⇧⌘S).*

**Підтримувані вхідні формати** — обов'язково **PNG, JPEG, WebP, TIFF, BMP і
GIF**. Цей список є чинним вхідним контрактом, а не прикладом: інші формати
відхиляються контрольовано. Зокрема, **HEIC/HEIF наразі навмисно не
підтримується** — файл HEIC/HEIF відхиляється як непідтримуваний формат.
Збереження відбувається у PNG, JPEG, WebP або TIFF (див.
[розділ 12](#12-крок-6--експорт)).

> **Максимальний розмір зображення: 40 мегапікселів.** Більші зображення
> відхиляються з повідомленням у рядку стану.

## 5. Панель інструментів (ліворуч)

Вертикальна панель уздовж лівого краю є **контекстною**: вона показує лише
інструменти активного на цей момент кроку. Зверху донизу:

### Переміщення / Масштаб (доступно завжди)

| Значок | Інструмент | Функція |
|---|---|---|
| ✥ | **Переміщення / Масштаб** | Перетягування лівою кнопкою зсуває вид, колесо миші масштабує. Активний на всіх кроках, крім *Вилучити фон* і *Рельєф і шари*. |

### Інструменти виділення (лише на кроці «Вилучити фон»)

| Значок | Інструмент | Функція |
|---|---|---|
| 🪄 | **Чарівна паличка** | Одним кліком виділяє суцільну область схожого кольору (заливка). Регулюється через *Допуск*. |
| 🖌 | **Пензель** | «Намалювати» виділення вручну. |
| 🧽 | **Гумка** | Видалити намальоване виділення. |
| ⬡ | **Полігональне ласо** | Клацайте точки по одній; **подвійний клік** замикає полігон. **Esc** скасовує. |

Швидке перемикання з клавіатури: **W** чарівна паличка, **B** пензель,
**E** гумка, **L** ласо – ці клавіші діють, лише поки активний крок
*Вилучити фон*.

Для всіх інструментів виділення:

- **Shift + клік** → **додати** до виділення
- **Ctrl/Cmd + клік** → **вирахувати** з виділення

### Інструменти висоти (лише на кроці «Рельєф і шари»)

| Значок | Інструмент | Функція |
|---|---|---|
| ▲ | **Підвищити (вище)** | Мазок піднімає висоту активного шару висот. |
| ▼ | **Знизити (нижче)** | Мазок опускає висоту активного шару висот. |

Обидва інструменти неактивні, доки немає активного шару висот (див.
[розділ 11](#11-крок-5--рельєф-і-шари)); тоді підказка пояснює причину. Вони
доповнюють дії підняти/опустити за повзунком у картковому інспекторі
довільним пензлем.

### Низ панелі: скасувати, повторити, тема

Внизу панелі інструментів – незалежно від кроку – залишаються видимими три
кнопки:

| Значок | Функція |
|---|---|
| ↩ | **Скасувати** (⌘Z) – відступити останній крок |
| ↪ | **Повторити** (⇧⌘Z) – застосувати скасований крок знову |
| ◐ | **Перемкнути світлу/темну тему** – змінює кольорову схему (та сама дія, що й `Вигляд → Світла тема`) |

> Видалення фону за допомогою ШІ, історія змін, а також відкриття/
> збереження зображення більше не розташовані на панелі інструментів: вони
> доступні через картковий інспектор відповідних кроків, меню або
> гарячі клавіші (див. [розділ 7](#7-крок-2--вилучити-фон) і
> [розділ 12](#12-крок-6--експорт)).

> **Порада:** наведіть курсор на значок, щоб побачити короткий
> підказковий текст (tooltip).

## 6. Виділення

Майже всі операції редагування (зробити прозорим, замінити колір)
застосовуються до **поточно виділеної області**. Виділення підсвічується
кольором на зображенні. Інструменти виділення активні на кроці *Вилучити
фон*.

![Завантажене зображення з активним виділенням](../../../app_screenshots/bgremover_complete_20260711_094027/02_main_loaded_selection.png)

*Завантажене зображення з активним виділенням: виділена область фону
підсвічена кольором на полотні.*

### За допомогою чарівної палички (рекомендовано для одноколірних фонів)

1. Виберіть чарівну паличку на панелі інструментів.
2. Клацніть по фону – всі схожі суміжні кольори буде виділено.
3. Недостатньо? Використайте **Shift+клік** для додавання ділянок або
   збільшіть **Допуск** (картка *Налаштування інструмента* на кроці
   *Вилучити фон*).

### За допомогою пензля та гумки (для точних коригувань)

- **Пензель:** малюйте по бажаній ділянці, щоб додати її до виділення.
- **Гумка:** малюйте по неправильно виділених ділянках, щоб видалити їх.
- **Розмір пензля** налаштовуйте на картці *Налаштування інструмента*.

### За допомогою полігонального ласо (для прямих країв)

1. Виберіть ласо.
2. Клацайте кут за кутом навколо об'єкта.
3. **Подвійний клік** замикає полігон і створює виділення.
4. **Esc** скасовує операцію.

## 7. Крок 2 – Вилучити фон

На кроці *Вилучити фон* ви відділяєте мотив від фону – автоматично за
допомогою ШІ або вручну. Картковий інспектор об'єднує для цього чотири
картки.

![Крок «Вилучити фон»](../../../app_screenshots/bgremover_complete_20260711_094027/11_step_2_cutout.png)

*Крок 2 «Вилучити фон»: угорі кнопка ШІ, нижче – налаштування інструмента,
дії з виділенням і «Редагувати фон».*

### Видалення фону за допомогою ШІ

Угорі в картковому інспекторі кнопка **«Вилучити фон (ШІ)»** видаляє фон
повністю автоматично. Під час першого виклику завантажується модель ШІ,
це може зайняти мить.

> Якщо компонент ШІ (`rembg`) не встановлено, кнопка неактивна.
> `Інструменти → Встановити видалення фону за допомогою ШІ…` показує
> потрібну команду встановлення для вашої платформи прямо в програмі
> (із кнопкою копіювання); альтернативно зверніться до посібника зі
> встановлення для налаштування функції ШІ.

Через `Інструменти → Керувати моделлю ШІ…` можна будь-коли перевірити,
чи модель ШІ вже завантажено, і почати або скасувати завантаження.

### Налаштування інструмента (допуск і розмір пензля)

| Повзунок | Діапазон | Ефект |
|---|---|---|
| **Допуск (чарівна паличка)** | 0 – 255 (за замовчуванням: 30) | Наскільки схожими мають бути кольори для спільного виділення чарівною паличкою. **Низько** = лише дуже схожі кольори · **Високо** = багато відтінків. |
| **Розмір пензля** | 4 – 200 пкс (за замовчуванням: 30 пкс) | Діаметр пензля та гумки. |

### Дії з виділенням

- **Зняти виділення** – скасовує поточне виділення. **Esc** спочатку
  скасовує активне обрізання або розпочате полігональне ласо й лише
  потім знімає виділення, якщо жодна з цих взаємодій не активна.
- **Інвертувати виділення** (⌘⇧I) – міняє місцями виділені й невиділені
  ділянки. Корисно: спочатку виділіть *об'єкт*, потім інвертуйте, щоб
  редагувати *фон*.
- **Розширити / Звузити** – збільшує чи зменшує виділення на радіус,
  заданий поруч (1 – 20 пкс, за замовчуванням: 2 пкс). Корисно для
  видалення тонкої кольорової смуги після вирізання.

### Редагувати фон

| Дія | Опис |
|---|---|
| **Вилучити (прозоро)** | Робить виділену ділянку повністю прозорою. Порада: спочатку виділіть фон чарівною паличкою. |
| **Вибрати колір** | Відкриває палітру кольорів. Маленька кольорова кнопка показує поточний колір заміни. |
| **Замінити колір** | Заповнює виділену ділянку вибраним кольором. |

![Діалог вибору кольору](../../../app_screenshots/bgremover_complete_20260711_094027/31_dialog_color_picker.png)

*Через «Вибрати колір» відкривається палітра; вибраний колір з'являється в
полі та застосовується до виділення кнопкою «Замінити колір».*

**Типовий робочий процес:** виділіть фон чарівною паличкою/ШІ →
*Вилучити (прозоро)* для вирізаного PNG, **або** виберіть колір і
*Замінити колір* для суцільного фону (наприклад, білого для фото на
документи).

### Згладити край (feather)

У розділі *Згладити край* тієї самої картки можна пом'якшити **альфа-край**
– корисно проти жорстких країв із виглядом «вирізаного» після вилучення
фону.

- **Радіус:** 0 – 20 пкс (за замовчуванням: 2 пкс) задає ширину м'якого
  переходу.
- **Згладити край** застосовує згладжування. Воно зачіпає лише **канал
  прозорості** (кольори не змінюються) і – коли активне виділення – лише
  всередині виділення.

## 8. Крок 3 – Корекція (кольору)

Крок *Корекція* містить просту **корекцію кольору**. Вона діє на **активний
кольоровий шар** (див. [розділ 11](#11-крок-5--рельєф-і-шари)) і залишає
прозорість незмінною.

| Повзунок | Діапазон | Ефект |
|---|---|---|
| **Яскравість** | 0 – 200 % (за замовчуванням: 100 %) | Освітлити або затемнити зображення. |
| **Контраст** | 0 – 200 % (за замовчуванням: 100 %) | Різниця між світлими й темними ділянками. |
| **Насиченість** | 0 – 200 % (за замовчуванням: 100 %) | Інтенсивність кольору; 0 % дає відтінки сірого. |

- Поки ви рухаєте повзунки, полотно показує **живий перегляд**.
- **Застосувати** фіксує корекцію (зі скасуванням/повторенням в історії).
- **Скинути** повертає всі три повзунки до 100 % і відхиляє перегляд.

## 9. Крок 4 – Форма й розміри

Крок *Форма й розміри* об'єднує поворот/дзеркалення, а також заокруглення
кутів, обрізання та швидку зміну розміру в пікселях.

![Крок «Форма й розміри»](../../../app_screenshots/bgremover_complete_20260711_094027/13_step_4_shape.png)

*Крок 4 «Форма й розміри»: поворот (швидкий поворот/довільний кут),
дзеркалення, заокруглення кутів, а внизу – формати обрізання.*

### Поворот

- **Швидкий поворот:** кнопки для *90° ліворуч*, *90° праворуч*, *180°* і
  *270°*.
- **Довільний кут:** повзунок або поле введення від **−180° до +180°**,
  далі **Застосувати кут**. На косих кутах виникають прозорі кути.

> Швидкий поворот доступний і з клавіатури: ⌘← (90° ліворуч) та ⌘→
> (90° праворуч).

### Дзеркало

- **Горизонтально** – відобразити ліворуч ↔ праворуч.
- **Вертикально** – відобразити вгорі ↕ внизу.

### Заокруглення кутів

1. Повзунком **Радіус** налаштуйте ступінь заокруглення (0 = без
   заокруглення, до 500 пкс = максимально кругло).
2. Натисніть **Заокруглити кути**.

Результат зберігається з прозорими кутами – найкраще у форматі PNG.

### Зміна розміру (пікселі, безпосередньо на кроці)

Картка «Змінити розмір» пропонує **ширину × висоту в пікселях**
безпосередньо на кроці: введіть значення й натисніть **Застосувати**. Для
пов'язаного співвідношення сторін, методу передискретизації та фізичних
розмірів (мм/DPI) скористайтеся повним діалогом із
[розділу 10](#10-зміна-розміру-та-фізичні-розміри).

### Вихідний формат і обрізання

1. Виберіть формат – на зображенні з'явиться **рамка**:
   - **Спеціальний формат:** ⬤ Коло
   - **Квадратний:** 1:1
   - **Альбомна орієнтація:** 16:9, 4:3
   - **Книжкова орієнтація:** 9:16, 3:4
2. **Перемістити рамку:** клацніть по центру та перетягніть.
3. **Змінити розмір:** перетягніть за кути – співвідношення сторін
   зберігається.
4. Над полотном з'являється панель:
   - **✓ Застосувати обрізання** – обрізає зображення.
   - **✗ Скасувати** – відкидає рамку.

![Активне кругле обрізання з панеллю підтвердження](../../../app_screenshots/bgremover_complete_20260711_094027/63_crop_circle_overlay.png)

*Приклад «Коло»: рамка обрізання з маркерами лежить над зображенням. Через
«✓ Застосувати обрізання» відбувається обрізання, «✗ Скасувати» відкидає
рамку.*

## 10. Зміна розміру та фізичні розміри

Через `Редагування → Змінити розмір…` (Ctrl+R) відкривається повний діалог
зміни розміру – з пов'язаним співвідношенням сторін, методом
передискретизації та фізичними розмірами. Для швидкої зміни розміру в
пікселях без діалогу на кроці *Форма й розміри* доступна вбудована картка
з [розділу 9](#9-крок-4--форма-й-розміри). Діалог підтримує дві одиниці
виміру:

### Зміна розміру в пікселях

У режимі **Пікселі** ви вводите **Ширину** та **Висоту** безпосередньо в
пікселях. З **Зв'язати співвідношення сторін** пропорція зберігається.
Метод передискретизації визначає якість:

| Метод | Придатність |
|---|---|
| **Ланцош** | Найкраща якість (за замовчуванням), ідеально для зменшення. |
| **Бікубічний** | Гладкі результати, добрий універсал. |
| **Білінійний** | Швидший, дещо м'якший. |
| **Найближчий сусід** | Зберігає жорсткі краї/пікселі, без згладжування. |

Діалог показує підсумкову кількість мегапікселів і дотримується ліміту в
**40 мегапікселів**.

### Фізичні розміри (мм/DPI) і площа друку

У режимі **Міліметри (мм + DPI)** ви задаєте **ширину/висоту в
міліметрах** і **роздільність (DPI)**; з них випливає розмір у пікселях.
Цей фізичний розмір є визначальним розміром друку й зберігається в проєкті
`.bgrproj`.

Через **Цільовий носій** ви вибираєте поширений носій друку (наприклад, A4
або A3). Якщо мотив на ньому вміщується, діалог це підтверджує; якщо він
більший за носій, підказка вказує на перевищення площі друку.

## 11. Крок 5 – Рельєф і шари

Крок *Рельєф і шари* об'єднує керування шарами й робочу область карти
висот у двох картках.

### Типи та ролі шарів

BgRemover може керувати кількома **шарами** в одному **проєкті** й
зберігати все як файл `.bgrproj`. Для класичного редагування фону вам не
потрібно цим займатися – одне зображення поводиться як єдиний кольоровий
шар. Кожен шар має **тип** і за бажанням **роль**. Лише **кольорові шари**
формують видиме кольорове зображення; решта типів – це шари даних для
підготовки друку.

| Тип / роль | Значення |
|---|---|
| **Колір** (кольоровий мотив) | Видиме зображення. Кілька кольорових шарів разом утворюють композит, який також експортується. |
| **Висота** (карта висот) | Карта висот у відтінках сірого для рельєфу/UV-друку. |
| **Глянець** (маска глянцю) | Маска для ефектів глянцю (експериментально). |
| **Загальний** | Нейтральний шар даних без фіксованої ролі. |

### Керування шарами

На картці *Шари* ви керуєте списком шарів:

| Дія | Опис |
|---|---|
| **Новий шар / Дублювати / Видалити** | Додати шар, скопіювати активний шар або видалити його. |
| **Вгору / Вниз** | Змінити порядок розташування шарів. |
| **Перейменувати** | Перейменувати активний шар. |
| **Роль** | Призначити роль активному шару (дозволені лише сумісні комбінації). |
| **Видимість** | Показати або сховати шар. |
| **Вибрати** | Вибрати шар як **активний** – інструменти діють на нього. |
| **Непрозорість** | Непрозорість шару (застосовується при відпусканні). |

### Файли проєктів (.bgrproj)

Через меню **Проєкт** ви працюєте з файлами проєктів:

- **Новий проєкт** (Ctrl+N), **Відкрити проєкт…** (Ctrl+Shift+O).
- **Зберегти проєкт** (Ctrl+Alt+S) та **Зберегти проєкт як…**
  (Ctrl+Alt+Shift+S).

Файл `.bgrproj` – це архів із **маніфесту** (порядок, типи, ролі, назви,
фізичні розміри) та **по одному PNG на шар**; шари висот додатково
зберігають свої 16-бітні значення висот в окремому файлі (версія формату
2). Так усі шари зберігаються без втрат – включно з прозорістю та
висотами в повній точності. Старі проєкти автоматично переймаються під
час відкриття та переводяться в новий формат під час наступного
збереження. Старіші версії BgRemover (до 2.6.0) не можуть відкрити
файли проєкту v2 і повідомляють про неочікуваний запис – сам файл
залишається незмінним. Проєкти також з'являються в
«Нещодавно відкритих» (див. [розділ 4](#4-крок-1--відкрити-зображення)).

### Карти висот: отримати

**Карта висот** – це шар у відтінках сірого, у якому яскравість
представляє висоту: **світле = високе, темне = низьке**. Вона є основою
для рельєфу та UV-друку. Картка *Висота* працює на активному **шарі
висоти**; розділи «Редагувати» та «Оптимізувати» активні лише тоді, коли
активний шар висоти.

- **Створити із зображення** – детерміновано перетворює поточне кольорове
  зображення на карту висот і створює її як новий шар висоти.
- **Імпортувати відтінки сірого…** – завантажує зображення у відтінках
  сірого як карту висот і масштабує його до розміру проєкту. 16-бітні
  файли у відтінках сірого (PNG/TIFF) імпортуються **нативно з усіма
  65536 рівнями**; кольорові та 8-бітні зображення перетворюються за
  їхньою яскравістю. 16-бітні зображення з альфа-каналом та
  float-зображення не можна прочитати без втрат – вони відхиляються з
  повідомленням. Під час експорту EufyMake програма попереджає, коли
  8-бітна ціль квантувала б внутрішні 16-бітні висоти.

### Карти висот: редагувати

- **Підвищити / Знизити** – піднімає або опускає висоту; **Сила** керує
  тим, наскільки сильно. Для довільного малювання на кроці *Рельєф і
  шари* додатково доступні однойменні інструменти-пензлі на панелі
  інструментів (див. [розділ 5](#5-панель-інструментів-ліворуч)).
- **Задати висоту** – встановлює висоту на фіксоване **значення**.
- **Інвертувати** – міняє місцями високе й низьке.

Коли активне виділення, дії за повзунками діють лише всередині
виділення, інакше – на весь шар.

### Карти висот: оптимізувати

Операції оптимізації показують **живий перегляд**; **Застосувати** фіксує
його (зі скасуванням/повторенням), **Відхилити перегляд** його відкидає.

| Операція | Ефект |
|---|---|
| **Рівні (чорний/білий)** | Встановити чорну й білу точку висоти. |
| **Гамма** | Підтягнути середні висоти світліше/темніше. |
| **Гаусове розмиття (радіус)** | М'яке рівномірне згладжування. |
| **Медіанне розмиття (радіус)** | Згладжує, зберігаючи краї. |
| **Поріг** | Розділити висоту на два рівні. |
| **Сходинки** | Квантувати висоту до певної кількості рівнів. |
| **Діапазон (мін/макс)** | Обмежити висоту діапазоном значень. |

## 12. Крок 6 – Експорт

Останній крок *Експорт* об'єднує 2D-перегляд, збереження зображення й
експорт для UV-друку у трьох картках.

### 2D-перегляд (колір, рельєф, висота, глянець, комбіновано)

**2D-перегляд** показує різні вигляди того самого мотиву безпосередньо на
полотні. Це **чисте екранне відображення**, яке не змінює ні зображення,
ні експорт. Картка *Перегляд* пропонує сегментований перемикач із чотирма
режимами; п'ятий режим «Комбіновано» доступний через `Вигляд → Режим
перегляду`.

| Режим | Відображення |
|---|---|
| **Колір** | Звичайне кольорове зображення. |
| **Рельєф** | Затінений рельєф із карти висот, накладений множенням на кольорове зображення. |
| **Висота** | Карта висот як зображення у відтінках сірого. |
| **Глянець** | Маска глянцю як глянцевий відблиск. |
| **Комбіновано** (лише через `Вигляд → Режим перегляду`) | Колір, рельєф і глянець разом. |

- За допомогою **Сили рельєфу** (0 – 100 %, за замовчуванням 70 %) ви
  задаєте інтенсивність рельєфу; при 0 % рельєф пропускається.
- **Показувати глянець** вмикає або вимикає складову глянцю.

Картка перегляду та підменю Вигляд залишаються синхронізованими. Невидимі
шари даних у перегляді ігноруються.

### Збереження

Картка *Збереження* пропонує вибір формату (PNG/JPEG/WebP/TIFF) і кнопку
збереження безпосередньо на кроці; альтернативно збережіть через меню:

- **Зберегти:** `Файл → Зберегти` (⌘S / Ctrl+S)
- **Зберегти як…:** `Файл → Зберегти як…` (⇧⌘S)

Під час збереження завжди записується **кольоровий композит** (незалежно
від того, який шар зараз активний або який режим перегляду встановлено).

| Формат | Властивості | Рекомендація |
|---|---|---|
| **PNG** | З прозорістю | Для вирізаних об'єктів – **рекомендований за замовчуванням** |
| **JPEG** | Без альфа-каналу, прозорі ділянки стають білими | Для фотографій з непрозорим фоном |
| **WebP** | Сучасний веб-формат, підтримує прозорість | Для використання в мережі |
| **TIFF** | Без втрат, підтримує прозорість | Для архівування/друку |

> Щоб зберегти вирізання, **завжди вибирайте PNG, WebP або TIFF** – JPEG
> заповнює прозорі місця білим кольором.

### Експорт для EufyMake Studio

Через картку *УФ-друк* на кроці *Експорт* або `Проєкт → Експортувати
ресурси для EufyMake Studio…` (Ctrl+Alt+E) BgRemover записує **ресурси
імпорту** для EufyMake Studio – **не** готовий файл `.empf`:

- **Кольоровий мотив** (обов'язково) як RGBA-PNG – із шару з роллю
  *Кольоровий мотив* або, якщо такого немає, із кольорового композиту.
- **Карта висот** (необов'язково) у відтінках сірого зі **світле = високе,
  темне = низьке** – доступна лише тоді, коли шар має роль *Карта висот*
  (наприклад, шар висоти, створений через «Створити із зображення»;
  звичайний шар висоти без цієї ролі не експортується).
- **Маска глянцю** (необов'язково, експериментально) як допоміжний
  ресурс – доступна лише тоді, коли шар має роль *Глянець*.

У діалозі ви вибираєте теку експорту, необов'язкові ресурси та **бітову
глибину** карти висот (8 біт за замовчуванням, 16 біт експериментально).
**Перевірка перед експортом** виконується безперервно й повідомляє про
висновки за рівнем серйозності:

- **Помилки** (⛔) блокують експорт до виправлення – наприклад, відсутній
  кольоровий мотив або невідповідні розміри.
- **Попередження** (⚠️) потрібно свідомо підтвердити – наприклад, порожні
  дані висоти/глянцю або непідтверджений 16-бітний вивід.

Далі ви імпортуєте та розташовуєте ресурси в EufyMake Studio, призначаєте
там режими фарби/шари й зберігаєте проєкт Studio самостійно як `.empf`.

## 15. Типові робочі процеси

### A) Вирізати фото товару (прозорий фон)

1. Відкрийте зображення.
2. На кроці *Вилучити фон* натисніть **«Вилучити фон (ШІ)»**.
3. **Гумкою**/**пензлем** доопрацюйте краї.
4. За потреби **звузьте** (1–2 пкс), щоб видалити кольорову смугу.
5. На кроці *Експорт* збережіть у форматі **PNG**.

### B) Фото на документи з білим фоном

1. Відкрийте зображення.
2. На кроці *Вилучити фон* клацніть **чарівною паличкою** по фону
   (налаштуйте допуск).
3. **Виберіть колір** (білий) → **Замінити колір**.
4. На кроці *Форма й розміри* виберіть формат **1:1**, розташуйте рамку,
   **✓ Застосуйте обрізання**.
5. На кроці *Експорт* збережіть у форматі **JPEG** або **PNG**.

### C) Кругле фото профілю

1. Відкрийте зображення.
2. Видаліть фон за допомогою **ШІ** (необов'язково).
3. На кроці *Форма й розміри* виберіть **⬤ Коло**, перетягніть рамку на
   обличчя.
4. **✓ Застосуйте обрізання**.
5. На кроці *Експорт* збережіть у форматі **PNG** (прозоро поза колом).

### D) Зберегти об'єкт, замінити лише фон

1. Відкрийте зображення, на кроці *Вилучити фон* клацніть **чарівною
   паличкою** по **об'єкту**.
2. **Інвертуйте виділення** (⌘⇧I) → тепер виділено фон.
3. Виберіть колір → **Замінити колір**.
4. На кроці *Експорт* збережіть.

### E) Ресурс рельєфу висоти для EufyMake Studio

1. Відкрийте зображення та вилучіть фон.
2. На кроці *Рельєф і шари* натисніть **Створити із зображення**.
3. Доопрацюйте висоту в розділі *Оптимізувати* (наприклад, *Рівні*,
   *Розмиття*) і натисніть **Застосувати**.
4. На кроці *Експорт* оберіть режим перегляду **Рельєф** або через
   `Вигляд → Режим перегляду` **Комбіновано** для перевірки.
5. Картка *УФ-друк* → перегляньте висновки й експортуйте.

## 16. Поради та хитрощі

- **Спочатку грубо, потім тонко:** вирізайте грубо за допомогою ШІ або
  чарівної палички, потім коригуйте пензлем/гумкою.
- **Налаштовуйте допуск:** якщо виділяється занадто багато → зменшіть
  допуск. Якщо занадто мало → збільшіть допуск або скористайтеся
  Shift+клік.
- **Видалення кольорової смуги:** після вилучення фону на кроці
  *Вилучити фон* застосуйте «Звузити» на 1–2 пкс, перш ніж видаляти фон.
- **М'які краї:** за допомогою *Згладити край* (крок *Вилучити фон*)
  вирізані краї виглядають менш жорсткими.
- **Крок назад:** кожен крок потрапляє в історію – через `Вигляд →
  Історія` подвійним кліком повертайтеся до будь-якого попереднього
  стану.
- **Нічого не виходить?** `Редагування → Відновити оригінал` повертає
  зображення до стану завантаження.

## 17. Відомі обмеження

- **Максимальний розмір зображення: 40 мегапікселів.** Більші зображення
  відхиляються.
- **Вхідні формати:** підтримуються PNG, JPEG, WebP, TIFF, BMP і GIF.
  **HEIC/HEIF наразі не підтримується** й відхиляється контрольовано.
- **Функція ШІ** вимагає необов'язкового компонента `rembg`. Без нього
  кнопка ШІ неактивна; усі ручні інструменти продовжують працювати.
- **2D-перегляд** є чистим екранним відображенням; експорт зображення
  без змін записує кольоровий композит.
- **Експорт EufyMake** створює лише ресурси імпорту, **а не** нативний
  файл `.empf`; 16-бітний вивід висоти є експериментальним.
- **Пакет програми** (`BgRemover.app`) є специфічним для macOS; у Linux
  програма запускається безпосередньо. Windows наразі не входить до
  офіційно протестованої матриці.

## 18. Усунення неполадок і файл журналу

У разі проблем варто перевірити внутрішній **файл журналу**
`bgremover.log`. Він зберігається у визначеній Qt директорії даних
програми й створюється під час першого запису в журнал. Точний шлях може
відрізнятися залежно від платформи й конфігурації Qt:

- **macOS (поточна конфігурація):**
  `~/Library/Application Support/BgRemover/BgRemover/bgremover.log`
- **Linux:** під `~/.local/share/`

Лаунчер App-бандла macOS додатково записує діагностику запуску в
`~/Library/Application Support/BgRemover/bgremover.log`.

Внутрішній файл містить повідомлення виконання й деталі помилок
(стек-трейси) і є першим місцем звернення при запитах підтримки.

Найпростіший спосіб знайти файл – через `Інструменти → Налаштування… →
Файл журналу`: там показується повний шлях, а кнопка **«Відкрити теку»**
відкриває директорію безпосередньо в менеджері файлів – зручно для
прикріплення файлу журналу до листа підтримки.

| Проблема | Можливе рішення |
|---|---|
| Кнопка ШІ неактивна | `rembg` не встановлено – `Інструменти → Встановити видалення фону за допомогою ШІ…` показує команду встановлення, або дивіться посібник зі встановлення |
| Зображення не відкривається | Понад 40 мегапікселів? Формат підтримується (не HEIC/HEIF)? Читайте рядок стану |
| ШІ працює дуже довго | Перший виклик завантажує модель – один раз, потім швидше |
| Прозорість зникла після збереження | Збережено у форматі JPEG → замість цього виберіть PNG/WebP/TIFF |
| Проєкт не відкривається | Пошкоджений/неповний файл `.bgrproj`? Читайте рядок стану |
